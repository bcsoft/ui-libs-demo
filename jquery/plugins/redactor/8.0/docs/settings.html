Settings


lang

Default setting is 'en'.

First you must download your language.

To setup a new language in Redactor you should do two things:

  • Connect the language file.
  • Call Redactor using the new language option.

For example (or see live example):

  1. <script src="redactor/langs/es.js"></script>
  2. <script type="text/javascript"> 
  3. $(document).ready(
  4.     function()
  5.     { 
  6.         $('#redactor').redactor({ lang: 'es' });
  7.     }
  8. );
  9. </script>

direction

Default setting is ltr

Sets text direction (see live example).

  1. $('#redactor').redactor({ direction: 'rtl' });

toolbar

Default setting is 'object'

If you want to turn a toolbar off set the toolbar option to false:

  1. $('#redactor').redactor({ toolbar: false });

buttons

By default, this setting contains the following array of toolbar's buttons:

  1. ['html', '|', 'formatting', '|', 'bold', 'italic', 'deleted', '|',
  2. 'unorderedlist', 'orderedlist', 'outdent', 'indent', '|',
  3. 'image', 'video', 'file', 'table', 'link', '|',
  4. 'fontcolor', 'backcolor', '|', 
  5. 'alignleft', 'aligncenter', 'alignright', 'justify', '|',
  6. 'horizontalrule']

If you wish to set your own array, set it in this option:

  1. $('#redactor').redactor({
  2.     buttons: ['html', '|', 'formatting', '|', 'bold', 'italic'] 
  3. });

buttonsAdd

If you need to add a new button to the toolbar, simply do the following:

  1. $('#redactor').redactor({
  2.     buttonsAdd: ['new_button'] 
  3. });

Or with divider:

  1. $('#redactor').redactor({
  2.     buttonsAdd: ['|', 'new_button'] 
  3. });

buttonsCustom

This option allows you to add your own buttons with callback to the toolbar.

For more information about creating your own buttons, go to Toolbar section.

  1. $('#redactor').redactor({
  2.     buttonsAdd: ['new_button'],
  3.     buttonsCustom: {
  4.         new_button: {
  5.             title: 'New Button', 
  6.             callback: function(obj, event, key) { … }
  7.         } 
  8.     }
  9. });

Or:

  1. $('#redactor').redactor({
  2.     buttons: ['html', '|', 'formatting', '|', 'bold', 'italic', '|', 'new_button'],
  3.     buttonsCustom: {
  4.         new_button: {
  5.             title: 'New Button', 
  6.             callback: function(obj, event, key) { … }
  7.         } 
  8.     }
  9. });

focus

Default setting is false.

This option allows you to set whether Redactor gets cursor focus on load or not.

  1. $('#redactor').redactor({ focus: true });

autoresize

Default setting is true.

This option turns on height autoresizing, which depends on the amount of text inputted into the text layer.

  1. $('#redactor').redactor({ autoresize: false });

fixed

Default setting is false.

If this option is turned on, Redactor's toolbar will remain at the top of the browser's window at all times.

  1. $('#redactor').redactor({ fixed: true });

convertLinks

Default setting is true

With this option turned on, Redactor will automatically replace URLs with hyperlinks.

  1. $('#redactor').redactor({ convertLinks: false });

autosave

Default setting is false

If you set a path to the handler file, Redactor automatically sends the code at default intervals. Autosave will send variable $_POST['data'] independently from textarea name.

  1. $('#redactor').redactor({ autosave: '/save.php' });

interval

Default setting is 60

Autosave interval (in seconds).

  1. $('#redactor').redactor({ autosave: '/save.php', interval: 30 });

imageGetJson

Default setting is false

A path to json file, which stores data about files that have already been uploaded.

  1. $('#redactor').redactor({ imageGetJson: '/images.json' });

json file example:

[
  { "thumb": "/images/1m.jpg", "image": "/images/1.jpg" },
  { "thumb": "/images/2m.jpg", "image": "/images/2.jpg" },
  { "thumb": "/images/3m.jpg", "image": "/images/3.jpg" },
  { "thumb": "/images/4m.jpg", "image": "/images/4.jpg" },
  { "thumb": "/images/5m.jpg", "image": "/images/5.jpg" }
]

imageUpload

Default setting is false

This option sets a path to the file that is responsible for uploading images. The handler will receive an array $_FILES['file'].

  1. $('#redactor').redactor({ imageUpload: '/image_upload.php' });

The handler should return JSON, for example:

  1. $array = array(
  2.     'filelink' => '/tmp/images/'.$file
  3. );
  4.  
  5. echo stripslashes(json_encode($array));

imageUploadCallback

Default setting is false

  1. $('#redactor').redactor({ 
  2.     imageUpload: '/image_upload.php', 
  3.     imageUploadCallback: function(obj, json) { … } 
  4. });

fileUpload

Default setting is false

This option sets a path to the file that is responsible for uploading files. The handler will receive an array $_FILES['file'].

  1. $('#redactor').redactor({ fileUpload: '/file_upload.php' });

The handler should return JSON, for example:

  1. $array = array(
  2.     'filelink' => '/files/'.$_FILES['file']['name'],
  3.     'filename' => $_FILES['file']['name']
  4. );
  5.  
  6. echo stripslashes(json_encode($array));

fileUploadCallback

Default setting is false

  1. $('#redactor').redactor({
  2.     fileUpload: '/file_upload.php', 
  3.     fileUploadCallback: function(obj, json) { … } 
  4. });

uploadCrossDomain

Default setting is false

This setting turns on/off drag and drop upload because it does not work for cross-domain upload.

  1. $('#redactor').redactor({ 
  2.     uploadCrossDomain: true
  3. });

uploadFields

Default setting is false

You can set up an hidden fields for upload (see live example).

The field value for "field1" will be taken from input by ID selector:

  1. $('#redactor').redactor({ 
  2.     uploadFields: {
  3.       'field1': '#field1'
  4.     }
  5. });

The field "field2" will automatically generated with value "12345".

  1. $('#redactor').redactor({ 
  2.     uploadFields: {
  3.       'field2': '12345'
  4.     }
  5. });

In the upload script you can take them from POST data.

overlay

Default setting is true

Turns on and off overlay for modal Redactor's windows.

  1. $('#redactor').redactor({ overlay: false });

callback

Default setting is false

Callback after an editor build.

  1. $('#redactor').redactor({ 
  2.     callback: function(obj) { … } 
  3. });

keyupCallback

Default setting is false

  1. $('#redactor').redactor({ 
  2.     keyupCallback: function(obj, event) { … } 
  3. });

keydownCallback

Default setting is false

  1. $('#redactor').redactor({ 
  2.     keydownCallback: function(obj, event) { … } 
  3. });

execCommandCallback

Default setting is false

This callback calls a function after any changes in Redactor by execCommand.

  1. $('#redactor').redactor({ 
  2.     execCommandCallback: function(obj, command) { … } 
  3. });

observeImages

Default setting is true

Enable or disable by clicking on the image to display a modal window image settings.

  1. $('#redactor').redactor({ 
  2.     observeImages: false
  3. });

air

Default setting is false

You can turn on an air-mode (see live example):

  1. $('#redactor').redactor({ 
  2.     air: true
  3. });

airButtons

Default setting is array

['formatting', '|', 'bold', 'italic', 'deleted', '|', 'unorderedlist', 'orderedlist', 'outdent', 'indent', '|', 'fontcolor', 'backcolor']

You can set up a custom buttons for air-mode:

  1. $('#redactor').redactor({ 
  2.     airButtons: ['formatting', '|', 'bold', 'italic', 'deleted']
  3. });

wym

Default setting is false

Turns on a visual structure mode (see live example):

  1. $('#redactor').redactor({ 
  2.     wym: true
  3. });

allowedTags

Default setting is array

["code", "span", "div", "label", "a", "br", "p", "b", "i", "del", "strike", 
"img", "video", "audio", "iframe", "object", "embed", "param", "blockquote", 
"mark", "cite", "small", "ul", "ol", "li", "hr", "dl", "dt", "dd", "sup", "sub", 
"big", "pre", "code", "figure", "figcaption", "strong", "em", "table", "tr", "td", 
"th", "tbody", "thead", "tfoot", "h1", "h2", "h3", "h4", "h5", "h6"]

You can set up another allowed tags on paste:

$('#redactor').redactor({ 
    allowedTags: ["a", "p", "b", "i"]
});

mobile

Default setting is true

This setting turns on/off the editor for mobile.

  1. $('#redactor').redactor({ 
  2.     mobile: false
